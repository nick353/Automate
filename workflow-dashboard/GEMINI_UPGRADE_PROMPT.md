# Gemini 3.0 Pro サイトアップグレードプロンプト

## 🎯 基本プロンプト（汎用・短縮版）

```
このワークフロー管理ダッシュボードサイトを、Awwwards.comに掲載されるレベルの圧倒的なクオリティでアップグレードしてください。

【現在のサイト】
- ワークフロー管理ダッシュボード（ブラウザ自動化タスクの管理）
- React + Tailwind CSS + Zustand
- ダーク/ライトモード対応
- レスポンシブデザイン

【参考サイトの雰囲気】
Linear.app、Vercel Dashboard、Notion、Zapier、Make.com、n8n、Retool、GitHub ActionsのUI/UXを参考に、それらを超える2025年のWebデザイントレンドを踏まえたプロフェッショナルなデザインにしてください。

【重要な要素】
- 没入感のある（Immersive）ユーザー体験
- エレガントで高級感のあるインターフェース
- スムーズなアニメーションとトランジション（フェードイン/アウト、スクロールアニメーション、ビュー・トランジション）
- グラスモーフィズムやニューモーフィズムの要素
- 視覚的な階層と余白の美しい配置

【創造性に任せる部分】
- 「このサイト、なんか好き」と無意識に感じる要素を分析し、それを実装してください
- 私が想像する以上の提案をしてください

【出力形式】
異なるアプローチのデザイン案を3パターン提示してください（例：ミニマル・エレガント、モダン・ダイナミック、プレミアム・高級感）
```

## 📋 詳細プロンプト（構造化版）

```
# ワークフロー管理ダッシュボード UI/UX アップグレード

## 目標
Awwwards.comに掲載されるレベルの圧倒的なクオリティで、2025年のWebデザイントレンドを踏まえたプロフェッショナルなダッシュボードにアップグレードしてください。

## 現在のサイト情報
- **種類**: ワークフロー管理ダッシュボード（ブラウザ自動化タスクの管理・監視）
- **技術スタック**: React 18 + Vite + Tailwind CSS + Zustand
- **主要機能**:
  - タスク一覧・作成・編集
  - リアルタイムライブビュー（CDP Screencast）
  - 実行履歴・ログ表示
  - 認証情報管理
  - ダーク/ライトモード切り替え
  - レスポンシブデザイン

## 参考サイト（雰囲気・レベル感）
- Linear.app（ミニマルでエレガント）
- Vercel Dashboard（モダンで機能的なUI）
- Notion（柔軟なレイアウト）
- Zapier（ワークフロー可視化）
- Make.com（リアルタイム実行の可視化）
- n8n（ノードベースUI）
- Retool（コンポーネント設計）
- GitHub Actions（シンプルで機能的なUI）

**これらを参考にしつつ、それらを超えるデザインを目指してください。**

## デザイン要件

### 視覚効果
- グラスモーフィズム、ニューモーフィズムの要素を取り入れる
- ノイズエフェクトや微細なテクスチャで質感を向上
- 美しいグラデーションとカラーパレット

### アニメーション
- パララックス（視差効果）
- スクロールアニメーション
- フェードイン/アウト
- ビュー・トランジション（滑らかなページ遷移）
- ホバーエフェクト
- ローディングアニメーション

### レイアウト
- 視覚的な階層の明確化
- 余白の美しい配置
- カードベースのデザイン
- サイドバーの洗練されたデザイン

### ユーザー体験
- 没入感のある（Immersive）インターフェース
- 直感的なナビゲーション
- リアルタイム更新の視覚的フィードバック
- エラーハンドリングのUI改善

## 創造性に任せる部分
- 「このサイト、なんか好き」と無意識に感じる要素を分析し、それを実装してください
- 私が想像する以上の提案をしてください
- ワークフロー管理ツールとして、ユーザーが「使いたくなる」デザインを考えてください

## 出力形式
以下の3パターンのデザイン案を提示してください：

1. **ミニマル・エレガント**: Linear風の洗練されたミニマルデザイン
2. **モダン・ダイナミック**: Vercel風のモダンで動的なデザイン
3. **プレミアム・高級感**: Notion風の高級感のあるデザイン

各案について：
- カラーパレット
- タイポグラフィ
- コンポーネントデザイン（カード、ボタン、フォームなど）
- アニメーションの提案
- レイアウトの改善点
- 具体的な実装方法（Tailwind CSSクラス、Reactコンポーネント構造）

## 技術的制約
- React 18 + Vite
- Tailwind CSS（カスタマイズ可能）
- Zustand（状態管理）
- 既存のコンポーネント構造を維持しつつ改善
- パフォーマンスを考慮した実装

## 優先順位
1. ユーザビリティの向上
2. 視覚的な魅力
3. パフォーマンス
4. アクセシビリティ
```

## 💡 使用のコツ

### 1. 段階的アプローチ
```
ステップ1: まず全体のデザインコンセプトを3案提示してもらう
ステップ2: 気に入った案を選んで深掘り
ステップ3: 具体的なコンポーネントごとに改善
```

### 2. イテレーション
```
「もっと余白を大きく取って」
「カードのホバーエフェクトを追加して」
「アニメーションをもっとスムーズに」
「この部分をもっと目立たせて」
```

### 3. 具体的な指示
```
「ダッシュボードの統計カードを、グラスモーフィズム風にして、ホバー時に軽く浮き上がるアニメーションを追加してください」
「サイドバーのナビゲーションアイテムに、アクティブ状態の美しいインジケーターを追加してください」
「実行履歴のテーブルを、よりモダンなカードレイアウトに変更してください」
```
